# @adjudicate/cli

> **Status: `0.1.0-experimental`** ‚Äî Pack lifecycle commands for policy authors.

The framework CLI for **authoring** Packs, complementing the operator
Console for **running** them.

```bash
# Scaffold a new Pack ‚Äî auto-detects monorepo vs standalone
adjudicate pack init pack-my-domain

# Validate a Pack against the kernel's conformance contract
adjudicate pack lint ./packages/pack-my-domain

# Verify the local environment
adjudicate doctor
```

---

## Detection model

The CLI auto-detects whether it's running inside a monorepo:

- **Monorepo** (a `pnpm-workspace.yaml` declaring `packages/*` is found
  walking up from cwd): `pack init` scaffolds under `<workspace-root>/packages/<name>`.
- **Standalone** (no workspace file found): scaffolds under `<cwd>/<name>`.

Override the target via `--target <dir>` on any scaffolding command.

This means the CLI works identically for:

- Framework contributors authoring inside this repo
- Enterprise platform teams adding internal Packs to their own monorepo
- Indie developers starting a single-Pack repository

The scaffolded Pack passes `assertPackConformance` immediately. Run
`pnpm test` in the new directory to verify.

### `adjudicate pack lint [path]`

Validates a Pack at the given path (defaults to cwd) against the kernel's
canonical conformance contract ‚Äî `assertPackConformance` from
`@adjudicate/core`. The CLI does not reimplement the rules; it asks the
kernel.

- ‚úó with the specific kernel-reported violation when it fails

## Pack #3 dogfooding

This CLI's own correctness is the prerequisite for the framework's
"Rule of Three" ‚Äî the next Pack (`pack-identity-kyc`, planned) will be
generated by `adjudicate pack init` itself, then customized for the
domain. If the scaffold can't pass conformance, the CLI is broken; if
KYC's customization clarifies a shared pattern with payments-pix, that
pattern becomes the seed of L2 primitive extraction.
